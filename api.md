# API Reference

The FastAPI backend provides several endpoints that the React Native mobile app consumes. Below is a detailed context of the available API routes.

## Base URL
The server runs locally, typically at:
`http://<your-local-ip-address>:8000`

Interactive Swagger documentation is automatically generated by FastAPI and is available at `/docs` (e.g., `http://localhost:8000/docs`).

## Core Endpoints

### 1. Chat API (`/chat`)
- **Description:** Main endpoint for text-to-text communication with the local LLM.
- **Method:** `POST`
- **Payload:** JSON containing the user's prompt and optional conversation history to maintain context.
- **Backend Flow:** The `chat.py` router receives the request and passes the prompt to `ollama_service.py`, which communicates with the local Ollama daemon to generate a response.

### 2. Voice API (`/voice`)
- **Description:** Endpoint for processing audio recordings from the mobile app (Speech-to-Text).
- **Method:** `POST`
- **Payload:** Multipart form data containing the audio file.
- **Backend Flow:** The `voice.py` router receives the file and uses `whisper_service.py` to transcribe the audio into text using a local Whisper model. It then usually forwards the transcribed text to the LLM for a response.

### 3. Text-to-Speech API (`/tts`)
- **Description:** Endpoint to convert AI text responses back into audio so the mobile app can play them aloud.
- **Method:** `POST`
- **Payload:** JSON containing the text string to be spoken.
- **Backend Flow:** The `tts.py` router processes the text and returns an audio file stream (e.g., WAV or MP3).

### 4. Vision API (`/vision`)
- **Description:** Endpoint for multimodal image analysis (e.g., analyzing skin conditions or uploaded health documents).
- **Method:** `POST`
- **Payload:** Multipart form data containing an image file, and optionally an associated text prompt.
- **Backend Flow:** The `vision.py` router passes the image and prompt to `ollama_service.py`, which utilizes a multimodal model (like `llava`) to analyze the image and generate a response.

### 5. Symptoms API (`/symptoms`)
- **Description:** Specialized endpoint for processing and structuring symptom logs.
- **Method:** `POST`
- **Payload:** JSON containing user-reported symptoms.
- **Backend Flow:** The `symptoms.py` router uses specific system prompts defined in `prompt_templates.py` to structure the unstructured symptom data into a defined JSON format for the mobile app's database.

### 6. Discovery API (`/discovery`)
- **Description:** Utility endpoints for the mobile app to locate the Edge Server on the local network.
- **Method:** `GET` / `POST`
- **Backend Flow:** The `discovery.py` router provides simple health checks (like a `/ping` endpoint) allowing the mobile app to verify connection status before sending large payloads.
